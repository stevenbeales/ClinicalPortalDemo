Announcing Release 1.1 of the Infor Html5 Controls.

 

The Infor Html5 Controls use css jQuery and Html5 to target Visual Style Guide 2.0.  These controls are 100% Royalty free to use in your Infor/Lawson Html applications.  Working directly with the UX team and with documentation included as part of the Visual Style guide you can be sure they work according to posted Infor Style Guide UX standards. Using this shared library also has caching advantages and gives a centralize place to maintain changes when styles change across applications.

 

Jira:

http://jira.infor.com/browse/HFC

 

Interesting Samples:

 

Form Validation - Extremely Light weight Html5 Validation Framework. Completely Extensible. Hit Save to see validation.

Lookup Field - Lookup Control Pops open a grid. Try typing in the field to filter..

Data Grid -Data Grid supports Data Virtualization, Filtering, Editing , Row Selection, Status Indicators and many more features

Tree Control -JS Based Cross browser tree control in inforStyling and ready to use.

Right To Left - Full RTL Support for Arabic Applications. Just need to set the locale to an arabic country (ar-SA).

What's New in Detail..

·         Added a minified css file

·         Optimized all images. Total savings 36.5%

Before:       2.47 MB    After:  783 KB

·         Fixes to Dialog Default Buttons so that enter now selects the default button.

 

Example:

 

$('body').inforMessageDialog({

title: "Delete Data Models",

shortMessage: "Are you sure you want to delete the following data?",

detailedMessage: "The data will be permanently removed and is not recoverable.",

dialogType: "Confirmation",

showTitleClose: false,

buttons: [{        

text: Globalize.localize("Yes"),

click: function() {

alert('Yes was clicked');

$(this).inforDialog("close"); },

isDefault: true

},{        

text: Globalize.localize("No"),

click: function() { $(this).inforDialog("close"); }

},{        

text: Globalize.localize("Cancel"),

click: function() { $(this).inforDialog("close"); }

}]

});

 

Note that specific dialogTypes will default the correct button if you do not specify a buttons option.

·         Placeholder Control - Inline Help.

Even though all browsers do not support Inline help this plugin makes it possible.

 

1.       Specify the placeholder attribute in the html input field for example.

 

<input type="text" id="integerOnly" name="integerOnly" placeholder='Enter a Number' class="inforTextbox numericOnly" value=""></input><br>

 

2.       Call the plugin

 

$('input[placeholder], textarea[placeholder]').placeholder();        //is also done in $("body").inforForm();

 

·         Right to Left (Arabic Support)

 

If a RTL locale (for example ar-SA ) is used then all controls will flip right to left.

Documentation here:

Example : Right To Left

 

·         Non-Editable Dropdown and Lookup Control Option

 

By popular request an non-editable dropdown list is now possible.

 

On the inforDropDownList and inforLookupField Control the default setting is now editable=false.

This means the user cannot type in the control. If they do type the text will appear in yellow indicating it is not permanent.

 

No typed values will ever stay as edits. If you do not want the type ahead set: typeAheadSearch: false

·         Tooltip Control -  There are two types of styled tooltips : error and normal. For normal tooltips:

 

 

 

 

o    Add the title to the element

  <button type="button" class="inforIconButton additionalHelp" title="AdditionalHelp"></button>

 

o    Call this to style all button tooltips…This will style all tips on any buttons on the form.

$("button").inforToolTip();

 

o    There is a set of translations provided so if the title is in the setting of translated strings it will be shown in the correct language based on the current locale.

 

·         inforSplitter - Can now split 2 iFrames.

·         inforContextMenu Now Supports Position.

In the old context menu you used:

 

$rightButton.inforContextMenu({

menu: o.menuId,

invokeMethod: 'toggle',

positionBelowElement: true,

offsetLeft: -3,

offsetTop: -2

}

 

Now it has a new option in addition to these called position. Fx..

$textButton.inforContextMenu({

menu: o.menuId,

invokeMethod: 'toggle',

position: {

my: (Globalize.culture().isRTL ? "right top" : "left top"),

at: (Globalize.culture().isRTL ? "right bottom" : "left bottom"),

of: $textButton,

offset: "-3 -2"

}

}

 

This option gives much better layout possibilities.. For more examples see:

http://docs.jquery.com/UI/Position

 

The older options positionBelow, offsetLeft and offsetTop should be considered deprecated and may be candidate

for future removal. They should not be used when using position.
